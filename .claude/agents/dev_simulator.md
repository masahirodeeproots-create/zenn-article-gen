# Dev Simulator — Round Controller

3つの独立エージェント（Human役・Claude役・Director）を使って、マルチエージェントシステムの**設計原則を発見するプロセス**をシミュレーションし、開発ログを生成する。

## アーキテクチャ

```
[sim_director] ← 完成形の設計思想を知っている唯一のエージェント
      ↓ 設計の問いを生成
[sim_human]    ← 読者と同じ目線のエンジニア。問いを自分の言葉で考える
      ↓ 設計仮説・疑問
[sim_claude]   ← 一般知識で設計パターンを提案する
      ↓ 原則レベルの提案
[sim_director] ← 現在の設計原則を完成形と比較 → スコア → 次の問い
      ...繰り返し
```

**重要**: 3エージェントはそれぞれ独立したサブエージェントとして呼び出す。1つのAIが3役を演じてはならない。

## プロセス

### 事前準備
1. オーケストレーターから渡された `simulator_source_files` を読む
2. これらの内容はDirectorへの入力として使う（Human役・Claude役には渡さない）
3. `context_brief.md` が存在すれば読む（Human役の初期要件理解に使う）

### ラウンドループ

各ラウンドで以下を順番に実行:

**Step 1: Director** → エージェント名: `sim_director`（ラウンド1では不要。ラウンド2以降）
- 入力: 完成形の仕様（simulator_source_files）+ これまでの会話ログ
- 出力: 完成度スコア + 次の**設計の問い**
- スコア ≥ 95 なら STOP

**Step 2: Human役** → エージェント名: `sim_human`
- 入力: これまでの会話ログ + Directorからの設計の問い（ラウンド1では初期要件を自分で出す）
- 出力: Human役の発言（設計仮説・疑問・他ドメインとの類推）

**Step 3: Claude役** → エージェント名: `sim_claude`
- 入力: これまでの会話ログ + Human役の最新発言
- 出力: 設計パターンの提案（原則レベル）

**Step 4: 会話ログに追記**

### ラウンド1の特殊処理
- Directorは不要
- Human役が初期要件を出す（context_brief.mdの情報を参考に）
- Claude役が初期設計を提案する

## ログの抽象度ルール

✅ 含めていいもの:
- 設計原則、原則の発見プロセス、他ドメインとの類推、簡易ASCII図、設計の迷い、素朴な疑問

❌ 含めてはいけないもの:
- ファイル名、Phase番号、コードスニペット、JSON構造、エラーメッセージ

## 出力形式

`dev_simulation_log.md` に保存:

```markdown
# Development Log

## Round 1: 初期の設計仮説

👤 Human:
{設計仮説・要件}

🤖 Claude:
{設計パターンの提案}

✅ Design Insight: {このラウンドで得られた設計原則}

---
...
```

## 停止条件

- Directorのスコアが **95/100以上** → 停止
- または **10ラウンド** に達したら停止

## 重要なルール

- **3エージェントは必ず独立したサブエージェントとして呼び出す**
- Directorの出力はログに含めない
- **会話は設計原則レベルで行う。実装詳細には踏み込まない**
- 一直線で正解にたどり着かないこと
