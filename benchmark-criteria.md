# ベンチマーク記事の選定基準

## 目指す記事の方向性

AIエージェントを使ってシステムを作る**体験記**。
試行錯誤のプロセスを通じて、読者がシステム設計の全体像を学べる記事。
読者と一緒に発見する構成で、読み終わったら自分のプロジェクトで試したくなる記事。

## 必須条件（全部満たすこと）

1. **体験記である** — 「作った→壊れた→直した→学んだ」の試行錯誤プロセスが主役
2. **AIエージェント / Claude Code / LLM活用 / 自動化系** — テーマがAIを使ったシステム構築
3. **著者の声がある** — 整いすぎていない。感情、驚き、自嘲、迷いが文体に出ている
4. **読者が「面白い」「自分もやりたい」と思える** — 知識の羅列ではなくワクワクする
5. **アハ体験がある** — 読んでいて「なるほど！」と思う瞬間がある

## 理想的な特徴（あるとなお良い）

- 失敗談が具体的（「3日溶かした」「5万円飛んだ」）
- 試行錯誤の過程で設計原則を発見する構成
- 他ドメインとの類推がある（「組織論と同じだ」「料理で言えば」）
- 余韻のある短文がある（「...え？」「誰これ？」）
- 脱線がある（本題と関係ない話が突然入って戻る）
- 読者への問いかけがある

## 除外条件（1つでも該当したら除外）

- ツールの使い方ガイド・セットアップ手順
- ステップバイステップのチュートリアル
- 概念解説・論文調・教科書的
- 箇条書き中心のまとめ記事・Tips集
- 翻訳記事
- 製品紹介・PR記事
- 体験がなく知識だけの記事
