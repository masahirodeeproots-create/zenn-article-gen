# Changelog — Iteration 1

## Style Guide 更新 (style_guide.md)

### Rule 0: 冒頭2段構成の精緻化
- **変更**: 「何の話か → フック」の固定順序を廃止し、フックの性質によって順序を選ぶ判断基準を追加
- **理由**: レビューで「普遍的に理解できるフック（薬機法アウトな表現が生成された）があるのにシステム説明を先行させた」と指摘。冒頭の没入が1〜2文遅延した
- **追加ルール**: フックに専門用語が不要であればフック先・文脈後。フックが専門用語依存なら文脈先・フック後
- **追加ルール**: 記事最大のインパクト数字（時間削減・コスト爆発）は冒頭フックか冒頭3段落内に出す

### Rule 14e（新規）: 連番見出しを使うなら全体を通せ
- **理由**: Round 1・2と立ち上げてRound 3・4を別セクション名に吸収、Round 5が再登場という不整合が発生。レビュワーが「Round 3はどこ？」と迷う構造的問題を指摘
- **内容**: 連番構造を使うなら全要素に見出しを与えるか、連番を一切使わないかの二択

## Anti-Patterns 追記 (anti_patterns.md)

### AP-001: 普遍的フックより先にシステム説明を置く
- 誰でも理解できるフックをシステム説明の後に置いた → 読者の没入が遅延
- スコアへの影響: 冒頭の引き込み力がベンチマーク比でやや劣ると評価

### AP-002: 強い数字を記事中盤に埋める
- 「45分→13分」「4,200円事故」が中盤以降初出 → 離脱読者に届かない
- スコアへの影響: タイトル・冒頭の引力がベンチマーク（「83%」）に劣ると評価

### AP-003: 連番セクション（Round N）の中抜け
- Round 3・4が連番見出しを持たず別名セクションに吸収 → 読者のナビゲーション体験が散漫
- スコアへの影響: 構成の揺れとして明示的に指摘

## Iteration 1 サマリー

- **Score**: 7.5/10
- **強み**: タイトル・冒頭フックの強度、Round形式の読者飽き防止、「嘘でしょ。」等の極短文
- **弱み**: 冒頭順序・数字前面化・連番一貫性の3点
- **次イテレーションの優先改善**: 冒頭を失敗エピソード先行に入れ替え、「45分→13分」を冒頭に引き上げ、Round 3・4の見出しを復活させるかRound番号を廃止する
