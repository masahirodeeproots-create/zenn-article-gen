# Review - Iteration 2

## 感想

前回から大幅に良くなっている。冒頭はそのまま強いし、「ベースのスープ」を核心に据えて深く掘る構成がはっきりしている。概念カタログ感はかなり減った。:::detailsに閉ループとモード切替を押し込んだのは良い判断。

ただ、読んでいて「この人の固有の体験」というより「よく整理された教材」に感じる部分がまだある。一番それを感じたのが類推の密度で、DDD、マイクロサービス、CI、MVCC、分散システム、製造業の工程間検査、フラクタル...と1記事に7つ以上の他ドメイン参照が出てくる。読者として「この人は本を読んで整理した」という印象を受けてしまう。ベンチマーク記事の著者たちは類推をほとんど使わず、自分の体験を直接語っている。

「嘘でしょ。」の極短文が入ったのは良い。感情の波が出てきた。ただ「これが一番腹が立った話です」と「嘘でしょ。」以外は温度が低い。全体的にもう少しだけ感情の上下が欲しい。

段落接続が「で、」「んで、」に偏っている。カジュアルさは出ているが単調。接続詞なしでいきなり次の話に入る段落がもっとあっていい。

## ベンチマークとの差分

**差分1: 類推の密度**

ベンチマーク「LLMonster 5万円」:
  "全部で 5 万円くらい気づいたら吹っ飛んだよ！" "Claude Codeを20万円分以上使いましたが"
  → 類推なし。自分の体験を数字付きで直接語る

生成記事:
  "DDDで言う「境界づけられたコンテキスト」" "分散システムの古典的なバグ" "データベースのMVCCと同じ発想" "製造業の工程間検査と同じ"
  → 1記事で7つ以上の類推。スタイルガイドでは2-3個まで

**差分2: 段落接続の多様性**

ベンチマーク「Claude Codeスキル攻略」:
  "残念！いらないものが出てきた。" "よし、原因が特定できた。"
  → 短い独立文。接続詞なし。テンポが良い

生成記事:
  "で、ここからが厄介な話なんですけど" "で、この「ダメ」が本当に厄介なのは" "んで、この「全体を先に確定させてから」"
  → 「で、」系接続が多い。ベンチマークのようなテンポの良い独立文がもっと欲しい

**差分3: 終わり方の余韻**

ベンチマーク「イルシル CRA→Vite」:
  泥臭い作業の記録がそのまま終わる。「きれいにまとめない」余韻がある

生成記事:
  "あなたのプロジェクトでエージェントを分割するとき" → 読者に直接語りかけるクロージングは悪くないが、やや「教育コンテンツ」感がある

## 次に直すべき1つ

**類推を3個以内に厳密に絞ること。** DDDの「境界づけられたコンテキスト」は記事の核心に直結するので残す。あと1つだけ許容。残りは全て自分の言葉で直接説明する。「〜と同じ構造です」「〜と同じ問題です」という文型を使うたびにカウントする。
