# AI研修課題 — 静的FBが難しいタスクのクオリティ自動改善

## 課題

> 【任意回答】「いいスライド」のような静的なFBが難しいようなタスクをひとつ設定し、そのクオリティの自動改善の方法を考案してください。

## 設定したタスク

**「Zennのトレンドに入るレベルの技術記事を自動生成する」**

技術記事の「良さ」はスライドと同様に静的なルールだけでは判定できない。
読者を引き込む冒頭、体験の生々しさ、トレンドとの接続、口語の自然さなど、人間が「面白い」と感じる要素は多次元で相互作用する。

## 考案した自動改善の方法

### リポジトリ

https://github.com/masahirodeeproots-create/zenn-article-gen

### アーキテクチャ概要

2つのPDCAサイクルを回すことで、「何を書くか」と「どう書くか」を分離して改善する。

```
素材PDCA（何を書くか）
  Phase 0: トレンド・読者の痛みをリアルタイム検索
  Phase 1: ソースコードから事実ベースの固定素材を生成
  Phase 2: 3独立AIで開発体験をシミュレーション → 素材改善ループ（最大5回）

記事PDCA（どう書くか）
  Phase 3: 全素材から記事生成 → ペルソナ記事と比較評価 → 改善ループ（最大10回）
```

### 「静的FBが難しい」問題への対処

| 課題 | 対処 |
|------|------|
| 「良い記事」の基準が曖昧 | Zennで実際にバズった記事11本をペルソナとして使い、相対評価する |
| チェックリスト型評価は機能しない | Reviewerに「率直な感想 + ペルソナ比較」を求め、人間の読者目線で評価させる |
| 素材の質と文体の質が混在する | 素材Reviewer（6基準で内容評価）と記事Reviewer（文体のみ評価）を分離 |
| 改善が収束しない | 停滞検出（±0.5×3回）と成功検出（2連続9.0以上）で自動停止 |
| 文体ルールが記事ごとに異なる | style_guide.mdを記事間で引き継ぎ、Consolidatorで圧縮して育てる |

### 12体のAIエージェント

| Phase | エージェント | 役割 |
|-------|------------|------|
| 0 | Trend Searcher | 5メディアからトレンド・痛みを検索 |
| 1 | Code Analyzer | ソースから固定素材5つを生成 |
| 2 | sim_human / sim_claude / sim_director | 3者シミュレーションで体験ストーリー生成 |
| 2 | Material Reviewer / Material Updater | 素材の評価・改善ループ |
| 3 | Writer | 全素材から記事生成 |
| 3 | Reviewer | ペルソナ記事と比較して文体評価 |
| 3 | Style Guide Updater / Consolidator | 書き方ルールの更新・圧縮 |
